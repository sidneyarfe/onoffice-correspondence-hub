# Story 3.9 — Autoatendimento logado: "Contratar novo produto" + "Minhas Assinaturas"

**Epic:** 003 — Motor de Receita Recorrente · **Fase:** 2 — Multi-produto
**Tipo:** UI · **Status:** Draft

> **Decisão (2026-06-24):** modelo **"1 produto por vez + add-on logado"**. Este é o **veículo
> principal de multi-produto** (carrinho público = 3.8, **deferida**).

## Contexto
A `cliente_planos` já suporta N assinaturas por cliente, mas `useClientPlanos` é **dead code** (nunca
importado) e `ClientPlanosModal` não toca a tabela. O cliente logado precisa ver suas assinaturas e
contratar produtos adicionais **reusando** seus dados, sem passar pelo funil público de novo.

## Acceptance Criteria
1. Seção **"Minhas Assinaturas"** no dashboard do cliente: lista as `cliente_planos` do cliente
   (produto, plano, `status`, `proximo_vencimento`, valor) — **reativando `useClientPlanos`** (read).
2. Botão **"Contratar novo produto"** → lista produtos/planos ativos **escondendo os que o cliente
   já tem ativos**.
3. Selecionar um plano cria **1 nova assinatura** via pipeline da Fase 1 (`processar-contratacao`),
   **reusando** os dados pessoais já cadastrados (sem redigitar) → gera contrato ZapSign + cobrança
   InfinitePay da nova assinatura.
4. Estado visual por `status` (`aguardando_assinatura` / `aguardando_pagamento` / `ativo` / `vencido`)
   com CTA contextual (assinar / pagar / —).
5. **`ui-ux-pro-max` antes da UI**; tokens dark premium (Epic 002); strings pt-BR; gates lint+tsc+build.

## Out of Scope
- Carrinho público multi-item (3.8, deferida). Admin (3.10). Upgrade/downgrade/cancelamento (Fase 3+).
