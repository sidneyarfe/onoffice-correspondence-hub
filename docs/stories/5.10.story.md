# Story 5.10 — Performance de navegação (sem recarregar) + Kanban de clientes por ciclo de vida

**Epic:** [005](./epic-005-modelo-dados-ideal-clientes.md) · **Fase:** Onda D
**Status:** Done (gates ✅)
**Dono:** @dev (Dex) · Orquestração: @aiox-master (Orion)

## Story
Como **admin**, quero que **trocar de tela não recarregue tudo** e que o **Kanban de Clientes reflita os
novos status de cliente** (ciclo de vida), para uma navegação fluida e uma visão correta da carteira.

## Acceptance Criteria
1. Trocar entre seções do admin **não pisca o spinner / não recarrega** os dados já vistos: cache em memória nos hooks de seção; remontagem inicializa do cache e atualiza em background. ✅
2. Cobertura: `useAdminClients`, `useProducts`, `useAdminCorrespondences`, `useAdminDocuments`, `useAdminData` (Dashboard). ✅
3. **Kanban da página Clientes** agrupa por **ciclo de vida** (Em ativação · Ativo · Inadimplente · Suspenso · Cancelado), com pílula de status no card; clique abre a ficha. ✅
4. Lifecycle por cliente **derivado em bulk** (todas as `assinaturas` + `faturas` vencidas) em `useAdminClients`. ✅
5. Funil de aquisição (leads) permanece **no CRM**, não no Kanban de clientes. ✅

## Dev Notes
- Causa do "recarrega ao trocar de tela": React Router desmonta a seção → hooks com `useState` refaziam fetch + spinner a cada remontagem. Corrigido com **cache de módulo** (sobrevive à remontagem); realtime/refetch seguem silenciosos.
- `QueryClient` existe no App mas estes hooks não o usavam — cache de módulo foi a correção de menor risco vs. reescrever 5 hooks para `useQuery` (registrado como débito).
- Kanban tornou-se **somente-leitura** (drag de funil removido); status muda pela ficha (com regras/senha). `onDropClient` mantido opcional para compat.
