## Story

Como equipe de desenvolvimento ON Office,
quero eliminar os bugs estruturais identificados na varredura (allowlist duplicada, JSX morto,
logout quebrado, dados falsos na UI),
para reduzir risco de regressão de segurança/UX e facilitar manutenção.

## Acceptance Criteria

1. Novo módulo `src/utils/adminEmails.ts` exporta `ADMIN_EMAILS`, e `isAdminEmail(email)`
   (case-insensitive, trim, aceita `undefined/null`); é a ÚNICA fonte da allowlist.
2. Os 11 pontos de duplicação passam a importar do módulo central: `AuthContext`,
   `ProtectedRoute`, `LoginPage`, `ForgotPassword`, `useAdminData`,
   `useAdminDataWithFallback`, `useAdminHealthCheck`, `useAdminCorrespondences`,
   `useCorrespondenceCategories`, `useDocuments`, `document-form/useDocumentFormLogic`.
   Comportamento equivalente preservado (allowlist canônica do CLAUDE.md).
3. `AdminOverview.tsx:151` — expressão morta substituída por renderização real e condicional
   de `<TempPasswordResync />` para o e-mail autorizado.
4. `AdminSidebar` — logout passa a usar `logout()` do AuthContext (sem chaves localStorage
   fantasmas, sem redirect para rota inexistente `/admin/login`).
5. `AdminHeader` — badge fake "5" e botão Settings sem ação removidos.
6. Handlers de logout (`ClientHeader`, `AdminHeader`) aguardam `logout()` (async) e não
   navegam manualmente (AuthContext já redireciona); interface `AuthContextType.logout`
   tipada como `() => Promise<void>`.
7. Lint + typecheck + build verdes; nenhum uso remanescente de allowlist inline
   (verificado via grep).
