## Story

Como equipe de desenvolvimento ON Office,
quero remover toda dependência e referência ao Lovable do build, da documentação e dos assets,
para que o projeto possa ser desenvolvido e implantado de forma totalmente autônoma.

## Acceptance Criteria

1. `lovable-tagger` removido de `package.json`/`package-lock.json` e do `vite.config.ts`;
   `npm run build` e `npm run dev` funcionam sem ele.
2. Logos servidos de caminho próprio (`public/brand/logo-light.png` / `logo-dark.png`) e
   `Logo.tsx` atualizado; nenhuma referência a `/lovable-uploads/` no código-fonte.
3. `index.html` sem assets externos de CDN do Lovable/gpt-engineer (favicon local).
4. `README.md` reescrito: setup local, scripts, stack, estrutura, Supabase, deploy genérico
   — sem qualquer menção operacional ao Lovable.
5. `CLAUDE.md` (raiz) atualizado: parágrafo "This is a Lovable project" substituído por
   nota de projeto independente (histórico preservado como origem).
6. Lint + typecheck + build verdes.
