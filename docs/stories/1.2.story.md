## Story

Como equipe de desenvolvimento ON Office,
quero que o client Supabase leia configuração de variáveis de ambiente e que o projeto esteja
preparado para a Supabase CLI (link, types, deploy de functions),
para ter acesso total e versionado ao banco sem depender do pipeline do Lovable.

## Acceptance Criteria

1. `src/integrations/supabase/client.ts` e `src/lib/api.ts` leem
   `import.meta.env.VITE_SUPABASE_URL` / `VITE_SUPABASE_PUBLISHABLE_KEY` com fallback para os
   valores atuais (não quebra builds sem .env). Nenhuma key inline duplicada em
   `callEdgeFunction`.
2. `.env.example` contém as variáveis `VITE_SUPABASE_*` documentadas.
3. `package.json` ganha scripts: `typecheck`, `db:link`, `db:types`, `db:push`,
   `functions:deploy`.
4. `docs/guides/supabase-cli.md` criado com passo a passo: login com a conta proprietária,
   link, regeneração de types, migrations e deploy de Edge Functions — incluindo o erro de
   privilégio atual e como resolvê-lo.
5. `.env` deixa de ser rastreado pelo git (`git rm --cached .env`) — arquivo local
   preservado; `.gitignore` já cobre.
6. Lint + typecheck + build verdes.
